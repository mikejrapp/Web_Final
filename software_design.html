Assigment 3 Software Design

Model

Game

The Game object controls the game state and logic

Game Variables

  • isGameOver: a boolean value which determines wheter the game has ended.
  • round: an integer value which keeps track of the rounds of the game.
  • turn: an intger value wich keeps track of player turns within a round.
  • turnOrder: an array the holds player objects in their turn order.
  • grid: a two dimensional array that holds the data for the "board" of the game.
  • gridSize: an integer value which determines the size of the grid (based on player choice).

Game Functions

  • initializeGame(player1,player2,size): this function sets the beginning of the game. It sets round and turn to 1 determines the turn order and sets up the grid.
  • reset(size): this function brings the game back to the first round keeping the turn order established by initializeGame.
  • setGrid(size): this function creates the two dimensional array that is the game grid.
  • setObstacles(size,i,j): this function is used with other functions to establish fixed obstacle tiles based on the grid size.
  • getGridSize(): this function returns the grid size.
  • getGridIndex(i,j): this function returns the value of stored in the grid array at index i and index j.
  • setTurnOrder(player1,player2): this function simulates a coin toss to determine which player goes first.
  • placeToken(character,indexI,indexJ): this function places a charcter in the grid array at the specified indexes.
  • updateGrid(character, pos): this function moves the specified character to the position pos in the array. pos is passed as a string which is parsed for the i and j indexes.
  • updateGameState(pGameOver,pRound,pTurn): this function updates the round and turn for the game as well as the isGameOver boolean.
  • combat(char1,char2,): this function handles character combat between char1 and char2.
  • spclAbl(char): this function handles a character using it's special ability. Toggling special abilities in the character object and updating the grid if applicable.

Player

Each game will have two player objects.

Player Variables

  • playerName: a string value that will have the players name.
  • playerNumber: an interger value that will represent whether the player is player 1 or 2
  • playerType: a string value that will determine if the player is a human or computer player
  • characters: an array which stores the players character objects.

Player Functions

  • addChar(character): this function will add a character object to the players list of characters.
  • getName(): this function will return the players name.
  • getPlayerNumber(): this function returns the players number.
  • getPlayerType(): this function returns the players type.
  • getCharacter(pos): this function will return a character object from the pos position in the characters list

Character

Each player will have at least one character that they control.

There are 4 classes of characters which inherit the character class

Each character has the following properties:

Character Variables

  • token: a string value which marks the character for use on the grid.
  • isDead: a boolean value which states whether the character is dead.
  • isStunned: a boolean value which states wheter the character is stunned.
  • healthPoints: an integer value for the characters health points.
  • actionPoints: an integer value that governs the amount of moves/actions a character can make in a turn.
  • attack: an interger value that determines the maximun damage a character can do in combat.
  • initiative: an interger value that determines when the character can move, attack or use a special ability during the players turn.
  • moveSpeed: an integer value that determines the amount of tiles a character can move in a turn.
  • attackRange: an integer value that determines how many tiles away from the characters current position they can attack.
  • currentHP: an integer value which has a maximum equal to healthPoints. If this number reaches 0 or less isDead will be set to true.
  • currentAP: an integer value that has a maximum equal to actionPoints. If this number reaches 0 or less the characters turn is over.
  • specialAbilities: an array that stores the characters special ability list as string values.
  • position: a string value that holds the characters current board position. The first number is the y position on the grid (or i index) and the second number is the x position (or j index).

Character functions

  • initChar(token,HP,AP,atk,init,mvSpd,atkRng,abl1,abl2): This function acts almost like a constructor. It is used to initialize a character object with the basic values needed. It does not change the characters board position.
  • setDeath(deathState): this function is used to toggle the isDead variable. If the characters currentHP variable is equal to 0 or less, this function will be called to set the isDead variable to true. It can also be used to "revive" a dead character by setting isDead to false.
  • setStun(stunState): this function is used to toggle the isStunned variable.
  • getHP(),getAP(),getAttack(),getInit(),getMvSpd(),getAtkRng(),getCurrentHP(),getCurrentAP(),getToken(): These all return their respective variable for use with other functions and objects.
  • getSpecialAbl(pos): this function will retrieve and return the special ability from the list at pos.
  • setPosition(indexI,indexJ): this function will set the characters board position to indexI (y value) to indexJ (x value).
  • getPosition(): this function will return the characters current board position.
